Setup gids — GSC Inspection API
Voor online marketeers · ±10 minuten eenmalige setup

Om de URL Inspection API te gebruiken heb je een Google Cloud Service Account nodig dat als eigenaar aan je Search Console property is toegevoegd. Volg de stappen hieronder.

1
Google Cloud Project aanmaken

Ga naar console.cloud.google.com en log in met het account dat toegang heeft tot Search Console. Maak een nieuw project aan via het project-dropdown bovenin → "Nieuw project".

🔗 console.cloud.google.com/projectcreate
2
Google Search Console API inschakelen

Ga naar API's & Services → Bibliotheek en zoek op "Google Search Console API". Klik op Inschakelen. Let op: dit is een andere API dan de Indexing API.

🔗 console.cloud.google.com/apis/library
→ Zoek: "Google Search Console API" → Inschakelen
3
Service Account aanmaken + JSON downloaden

Ga naar API's & Services → Inloggegevens → Inloggegevens aanmaken → Service-account. Geef het een naam. Ga daarna naar het tabblad Sleutels → Sleutel toevoegen → JSON. Sla het bestand veilig op.

🔒Bewaar dit bestand veilig — deel het nooit via e-mail of in gedeelde mappen.
4
Service Account toevoegen als Eigenaar in GSC

Ga naar search.google.com/search-console → je property → Instellingen → Gebruikers en machtigingen → Gebruiker toevoegen. Vul het service account e-mailadres in en geef het de rol Eigenaar.

📧 naam@project.iam.gserviceaccount.com
🔑 Rol: Eigenaar — verplicht voor Inspection API!
⚠️De rol Eigenaar is verplicht. "Volledige gebruiker" geeft geen toegang tot de Inspection API.
5
Site URL correct invullen

De Site URL moet exact overeenkomen met hoe je property in Search Console staat. Open GSC en kopieer de URL precies zoals hij bovenin staat.

URL-prefix property: https://www.jouwsite.nl/
Domain property: sc-domain:jouwsite.nl
⚠️Een kleine afwijking (wel/geen www, wel/geen trailing slash) geeft een fout. Kopieer de URL letterlijk uit GSC.
6
Resultaten interpreteren

De tool toont per URL de exacte status die Google teruggeeft:

✓ Geïndexeerd → URL staat in de Google-index
⚠ Niet geïndexeerd → Google kent de pagina maar indexeert hem niet
✗ Onbekend → Google heeft de pagina nog nooit gezien

Bij "Niet geïndexeerd" zie je de reden (bijv. Crawled - currently not indexed of Excluded by noindex). Dit helpt je te bepalen welke actie je moet ondernemen in Search Console.

✅ Pagina's met status "Niet geïndexeerd" kun je gericht handmatig indienen via de URL-inspectietool in GSC — dat geeft de snelste resultaten.
Veelgestelde vragen
Wat is het verschil met de vorige Indexing API?
De oude Indexing API stuurde een signaal naar Google om een URL te indexeren — maar dat signaal werd buiten JobPosting/BroadcastEvent pagina's grotendeels genegeerd. De Inspection API geeft je terugkoppeling over de werkelijke status en de reden. Zo weet je wat er echt aan de hand is.
Wat is de limiet van de Inspection API?
2.000 verzoeken per dag per Google Cloud project. Aanzienlijk meer dan de Indexing API. De tool houdt dit bij in je browser.
Kan ik ook via deze tool indexering aanvragen?
Nee — er bestaat geen publieke API om de "URL inspecteren → Indexering aanvragen" knop in GSC te simuleren. Die knop gebruikt een interne Google-route. De snelste werkende aanpak blijft: gebruik de resultaten van deze tool om te zien welke pagina's aandacht nodig hebben, en dien die vervolgens handmatig in via GSC.
Worden mijn gegevens opgeslagen?
Nee — credentials en URL's verlaten nooit je browser. Geen server, geen database.
