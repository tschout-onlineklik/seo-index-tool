Google Search Console · Inspection + Indexing API

Deze tool combineert twee Google APIs. Fase 1 inspecteert de huidige indexeringsstatus via de URL Inspection API. Fase 2 dient alle URL's in via de Indexing API om Google te vragen ze te (re)crawlen.

1
Google Cloud Project + beide APIs inschakelen

Maak een project aan op console.cloud.google.com. Schakel vervolgens twee APIs in via Bibliotheek:

1. "Google Search Console API" → voor inspecteren
2. "Web Search Indexing API" → voor indienen
2
Service Account aanmaken + JSON downloaden

Ga naar Inloggegevens → Service-account aanmaken. Download daarna een JSON sleutelbestand via Sleutels → Sleutel toevoegen → JSON.

🔒Bewaar dit bestand veilig — het geeft volledige API-toegang.
3
Service Account toevoegen als Eigenaar in GSC

Ga naar Search Console → je property → Instellingen → Gebruikers → Gebruiker toevoegen. Vul het service account e-mailadres in en geef de rol Eigenaar.

📧 naam@project.iam.gserviceaccount.com
🔑 Rol: Eigenaar (verplicht voor beide APIs)
4
Site URL correct invullen

Kopieer de property URL letterlijk uit Search Console, inclusief trailing slash.

URL-prefix: https://www.jouwsite.nl/
Domain: sc-domain:jouwsite.nl
Kan ik een ban of penalty krijgen?
🔍 URL Inspection API — volledig veilig
De Inspection API is door Google gebouwd voor precies dit gebruik: bulk-inzicht in indexeringsstatus voor jouw eigen property. Lezen geeft nooit een penalty. Geen enkel risico.
📤 Indexing API — veilig, maar met nuance
Google geeft een HTTP 200 terug op elk verzoek — ook voor reguliere pagina's. Er is geen technische beperking die het blokkeert. De API is officieel bedoeld voor JobPosting en BroadcastEvent pagina's, maar er staat geen straf op het gebruik voor andere pagina's. Google kan het signaal simpelweg negeren of lager prioriteren. Een ban of penalty krijg je hier niet van — dat zijn sancties voor manipulatie van zoekresultaten (spam, cloaking, gekochte links), niet voor API-gebruik voor je eigen site.
Wat als Google de Indexing API verzoeken negeert?
Dan verandert er niets — je bent precies waar je was. Het slechtste wat kan gebeuren is dat het niet werkt. De beste uitkomst is dat Google de pagina's prioriteit geeft in de crawlwachtrij. Voor pagina's die "Onbekend bij Google" zijn, helpt de Indexing API weinig: die pagina's zijn niet vindbaar via interne links of een sitemap. Die moet je eerst toegankelijk maken.
Wat zijn de quota's?
Inspection API: 2.000 verzoeken per dag per property. Indexing API: 200 verzoeken per dag (standaard). Als je meer nodig hebt, kun je bij Google een quota-uitbreiding aanvragen via de Google Cloud Console.
